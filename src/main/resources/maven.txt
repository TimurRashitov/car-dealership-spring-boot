Lifecycle (Этапы жизни или циклы):

clean удаляет результат предыдущей сборки типично clean package

validate быстрая проверка что проект валидный(проверяется структура проекта), есть валидный pom и базовая структура

compile быстрая проверка компиляции кода; src/main/java --> target/classes (компиляция идет в папку)

test запускает unit-тесты типично перед package

package собирает артефакт проекта и кладет его в target

verify проверка качества сборки, валидация артефактов, плагины контроля

install кладет артефакт проекта в локальный репозиторий

site документация и отчеты

deploy публикует артефакт(проект) на удаленном репозитории, делает релизную версию. Альтернативы: Jenkins, Tekton

Команды для терминала:
.\mvnw clean                    -очищает и убирает папку target
mvn clean                       -очищает и убирает папку target
mvn clean package -DskipTests   -пропрыгивается (пропускается) стадия тестов один раз
mvn dependency:tree             -вывод на экран дерева зависимостей
